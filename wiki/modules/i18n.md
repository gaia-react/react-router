# i18n

GAIA uses [[remix-i18next]] (built on i18next) with **TypeScript files** instead of JSON for translations.

## TypeScript over JSON

> [!key-insight] TS language files compose
> Most templates use JSON. GAIA uses `.ts` files because:
>
> - ESLint + Prettier keep them clean
> - TypeScript catches typos and renamed keys
> - Auto-completion in editors
> - You can `import` strings into tests
> - Dot-chain composition: `pages.account.profile.fullName`
>
> Tradeoff: all strings load up front (per language). In practice, gzip + caching makes this a non-issue. Switch to JSON later if you need on-demand loading.

## Folder layout

```
app/languages/
├── index.ts        # imports all languages (`en`, `ja`)
├── en/
│   ├── index.ts    # imports root files
│   ├── common.ts
│   ├── errors.ts
│   └── pages/
│       ├── index.ts        # imports sub-pages
│       ├── _index.ts       # root index page strings
│       └── legal.ts
└── ja/             # same structure
```

> [!note] `_index.ts` vs `index.ts`
> `index.ts` imports sub-pages. `_index.ts` holds strings for the root of that page. Usually `_index` is destructured (`...index`); for the **actual** index page of the site, it's named (`{index}`).

## Usage in components

```tsx
const {t} = useTranslation('pages', {keyPrefix: 'index'});
return <h1>{t('title')}</h1>;
```

## Rules

- **One `useTranslation()` call per component** — destructure `t` (and `i18n` if needed)
- **Use `keyPrefix`** for nested keys: `{keyPrefix: 'index'}` then `t('title')` instead of `t('index.title')`
- **Check `common.ts`** before duplicating shared strings (Save, Cancel, Delete, Close)
- **Namespaces**: `common`, `pages`, `errors`

See [[Coding Guidelines]] § translation rules for the strict version.

## Server-side i18n in loaders

For meta tags, use `getInstance()` from the i18next middleware:

```ts
const i18next = getInstance(context as RouterContextProvider);
const title = i18next.t('index.meta.title', {ns: 'pages'});
```

The middleware lives in `app/middleware/i18next.ts` and is registered in `root.tsx`:

```ts
export const middleware = [i18nextMiddleware];
```

## Adding a key

1. Determine namespace (`common`, page-specific, `auth`, `errors`)
2. Add to `app/languages/en/{file}.ts`
3. Add to **all** other language files
4. Use nested objects for related keys

The `/gaia-init` command lets you pick languages on initial setup. The `/new-route` command auto-creates i18n keys.

## Storybook

[Storybook-react-i18next](https://github.com/stevensacks/storybook-react-i18next) addon is wired up so you can switch languages in Storybook. Strings load from `.storybook/i18next.ts`, applied in `preview.ts`.

## Type safety

Type-safe keys + lint-staged means you cannot commit code referencing a missing key.
