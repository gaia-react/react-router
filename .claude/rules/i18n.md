# Internationalization (i18n)

## Usage

Use `t()` from `useTranslation()` for all user-facing strings. Never hardcode text in JSX.

```tsx
const {t} = useTranslation('pages', {keyPrefix: 'index'});
return <h1>{t('title')}</h1>;
```

## Rules

- **One `useTranslation()` call per component** — destructure `t` (and `i18n` if needed)
- **Use `keyPrefix`** for nested keys to avoid repetition: `{keyPrefix: 'index'}` then `t('title')` instead of `t('index.title')`
- **Check `common.ts` before duplicating** — shared keys (Save, Cancel, Delete, Close, etc.) belong in common namespace
- **Namespace convention**: `'common'` for shared, `'pages'` for page-specific, `'errors'` for error messages

## Language Files

All files under `app/languages/en/`:

- `common.ts` — shared keys (form labels, buttons, generic terms; includes `email`, `emailPlaceholder`, `password`)
- `errors.ts` — error messages
- `pages/` — page-specific keys organized by route
  - `pages/_index.ts` — index page

## Adding New Keys

1. Determine namespace: common, page-specific, or errors
2. Add key to the English file under `app/languages/en/`
3. Add to ALL other language files (check `app/languages/` for supported locales)
4. Use nested objects for related keys

## Server-Side i18n (Loaders)

For meta tags in loaders, use `getInstance()` from middleware:

```ts
const i18next = getInstance(context as RouterContextProvider);
const title = i18next.t('index.meta.title', {ns: 'pages'});
```
